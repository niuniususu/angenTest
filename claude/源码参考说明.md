# Claude Code 泄露源码本地参考说明

---

## 一、背景

2026 年 3 月 31 日，Anthropic 发布 `@anthropic-ai/claude-code` v2.1.88 到 npm 时，因 `.npmignore` 遗漏导致 `cli.js.map`（59.8 MB source map）被打包。该 map 文件引用了 Cloudflare R2 存储桶中的 `src.zip`，桶未设认证，任何人可下载完整源码。

泄露在约 2 小时后被 Anthropic 修复，但源码已被大量镜像和 Fork。Anthropic 通过 DMCA 下架了 8,100+ 个 GitHub 仓库，但 clean-room 重写版和去中心化镜像至今仍存。

本目录保存了两份不同定位的源码参考，供课程制作时查阅验证。

## 三、两份源码的详细说明

### 3.1 collection（综合参考版）

**来源**：[github.com/chauncygu/collection-claude-code-source-code](https://github.com/chauncygu/collection-claude-code-source-code)

**定位**：一个精心整理的"学习资料合集"，包含原始源码 + 反编译版 + 重写参考 + 分析文档。适合全景浏览和交叉对比。

**核心内容**：

| 子目录 | 内容 | 说明 |
|--------|------|------|
| `original-source-code/original-src/src/` | v2.1.88 原始 TypeScript 源码 | 1,884 个文件，512,664 行。从 npm source map 中的 `src.zip` 直接解压，未经修改 |
| `claude-code-source-code/` | 反编译版 + 补充 | 含 stubs/（Anthropic 内部包的占位实现）、补充文档 |
| `claw-code/` | Python clean-room 重写 | 来自 Sigrid Jin 的 claw-code 项目骨架 |
| `nano-claude-code/` | 极简 Agent 实现 | ~200 行 Python 的最小可运行 Agent |
| `docs/` | 社区分析文档 | 含 PDF 格式的深度架构分析 |

**使用场景**：
- 查看原始源码的某个模块实现 → 进入 `original-source-code/original-src/src/`
- 对比原始版和重写版的差异 → 交叉查看 `original-source-code/` 和 `claw-code/`
- 阅读社区分析 → 查看 `docs/`

### 3.2 buildable-fork（可构建版）

**来源**：[github.com/beita6969/claude-code](https://github.com/beita6969/claude-code)

**定位**：在原始泄露源码基础上，逆向重建了完整的构建系统（`bunfig.toml`、`package.json`、`tsconfig.json`），使代码可以用 Bun 实际构建运行。

**特点**：
- 可用 `bun install && bun run src/entry.ts` 构建运行（需要 Bun >= 1.3.x）
- 补齐了 60+ npm 依赖声明
- 对 90+ 个 Anthropic 内部包做了 stub 处理（功能受限但结构完整）
- 1,439 个 .ts/.tsx 文件，512,913 行

**使用场景**：
- 需要实际运行某段代码验证行为 → 使用此版本
- 需要通过 IDE 跳转查看类型定义和模块关系 → 使用此版本（构建配置完整，IDE 支持更好）

**局限**：
- 90+ 个功能模块因依赖内部包而只有 stub 实现
- 100+ 个 Anthropic 内部包（如 `@anthropic-ai/auth`）为最小占位
- 实际可运行程度有限，但代码结构和逻辑完整可读

## 五、关于代码行数的口径说明

知识库调研报告中多处引用"QueryEngine.ts 约 46,000 行"，但实际源文件 `QueryEngine.ts` 为 1,295 行。这一差异的原因是：

- **46,000 行**：来自社区对 bundled 产物（`cli.js.map` 中 source map 映射的合并代码段）的统计，包含了 QueryEngine 及其所有依赖模块的合并行数
- **1,295 行**：是 `QueryEngine.ts` 单个源文件的真实行数

两个数字都是"正确"的，只是统计口径不同。课程中引用时建议说"QueryEngine 及其关联模块群"或"推理引擎相关代码"，而非特指单个文件。

同理，"51.2 万行"是所有 .ts/.tsx 文件的总行数（512,664 行），经实际验证与各方报道一致。

## 六、法律与使用注意事项

1. **本目录仅用于课程准备阶段的技术事实验证**，不在课件中直接展示原始泄露代码
2. 课件中使用 learn-claude-code 的开源代码（MIT 许可）+ 架构图 + 伪代码讲解
3. 课件开头声明课程定位为"架构拆解与学习"
4. 已加入 `.gitignore`（规则：`knowledge-base/*/源码参考/`），不会提交到仓库
5. 如需引用某段代码的设计思路，转化为架构图或伪代码后使用，避免直接引用原始代码
